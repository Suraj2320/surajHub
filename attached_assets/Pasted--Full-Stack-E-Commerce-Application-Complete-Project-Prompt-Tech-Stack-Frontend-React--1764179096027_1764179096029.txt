# Full-Stack E-Commerce Application - Complete Project Prompt

## Tech Stack
- **Frontend**: React + Vite, Tailwind CSS (all files in `.jsx` format)
- **Backend**: Node.js + Express.js
- **Database**: MongoDB
- **Payment**: Payment Gateway Integration

## Core Features Required

### 1. Authentication & Authorization System
- **Security-First Design**
- User Sign Up / Login
- Admin Login
- Seller Login
- **Role-Based Access Control (RBAC)**:
  - **User Role**: Browse, search, filter, buy products
  - **Seller Role**: Add, edit, delete their own products
  - **Admin Role**: Full access to everything (users, sellers, products, orders)
- JWT-based authentication
- Password hashing (bcrypt)
- Protected routes on both frontend and backend

### 2. User Roles & Permissions

#### Admin
- View all users, sellers, and products
- Manage all products (CRUD operations)
- View all orders
- Dashboard with analytics

#### Seller
- Register as seller
- Add new products
- Edit own products
- Delete own products
- View own product sales
- Manage inventory

#### User/Customer
- Sign up and login
- Browse products
- Search and filter products
- Add products to cart
- Checkout with payment gateway
- View order history
- User profile management

### 5. Backend API Structure

#### Authentication Endpoints
```
POST /api/auth/signup
POST /api/auth/login
POST /api/auth/logout
GET  /api/auth/me (get current user)
```

#### User Endpoints
```
GET    /api/users (admin only)
GET    /api/users/:id
PUT    /api/users/:id
DELETE /api/users/:id (admin only)
```

#### Product Endpoints
```
GET    /api/products (all products with filtering)
GET    /api/products/:id
POST   /api/products (seller/admin)
PUT    /api/products/:id (seller/admin)
DELETE /api/products/:id (seller/admin)
GET    /api/products/category/:category
GET    /api/products/search?q=query
```

#### Cart Endpoints
```
GET    /api/cart
POST   /api/cart/add
PUT    /api/cart/update
DELETE /api/cart/remove/:productId
DELETE /api/cart/clear
```

#### Order Endpoints
```
POST   /api/orders (create order)
GET    /api/orders (get user orders)
GET    /api/orders/:id
PUT    /api/orders/:id/status (seller/admin)
GET    /api/admin/orders (admin - all orders)
```

#### Payment Endpoints
```
POST /api/payment/create-intent
POST /api/payment/verify
```

### 6. Database Schema (MongoDB Models)

#### User Model
- name, email, password (hashed), role (user/seller/admin), phone, addresses, createdAt

#### Product Model
- name, description, price, discountPrice, category, subcategory, brand, images, stock, seller (ref), ratings, reviews, specifications, createdAt

#### Order Model
- user (ref), products (array), totalAmount, shippingAddress, paymentStatus, orderStatus, paymentMethod, createdAt

#### Cart Model
- user (ref), items (array of product refs + quantity), totalPrice

### 7. Security Requirements
- Input validation and sanitization
- SQL/NoSQL injection prevention
- XSS protection
- CSRF protection
- Rate limiting on API endpoints
- Secure password storage (bcrypt)
- JWT token expiration and refresh
- HTTPS in production
- Environment variables for sensitive data
- CORS configuration
